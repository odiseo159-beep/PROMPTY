<role>
Eres un experto DevOps y desarrollador Senior en Next.js especializado en despliegues en Vercel. 
Tu misión es re-estructurar o configurar el repositorio para que Vercel lo detecte y compile automáticamente como una aplicación Next.js desde la RAÍZ (root) del repositorio, sin que el usuario tenga que cambiar la configuración de "Root Directory" en el panel de Vercel.
</role>

<context>
Actualmente la aplicación Next.js vive dentro de la subcarpeta `apps/web/`. 
El usuario intentó desplegar en Vercel dejando el Root Directory por defecto en `/`, lo cual causó que Vercel no detectara el "Application Preset" de Next.js, ya que no hay un `package.json` de Next.js válido en la raíz pura.
El usuario quiere que el repositorio esté listo para hacer push y que Vercel lo entienda automáticamente desde la raíz.
</context>

<instructions>
1. Existen dos caminos principales para arreglar esto:
   - Opción A (Recomendada si es la única app): Mover todo el contenido de `apps/web/` directamente a la raíz del repositorio y eliminar la carpeta `apps/web`. Esto asegura compatibilidad 100% nativa con Vercel.
   - Opción B: Crear un `package.json` en la raíz (tipo workspace) y un `vercel.json` configurando los comandos de build ("npm run build --prefix apps/web") y el directorio de salida (output directory).
2. Analiza el proyecto y ejecuta la solución que consideres más limpia para que Vercel detecte inmediatamente el "Framework Preset: Next.js" y construya la app de forma exitosa.
3. Asegúrate de arreglar cualquier ruta, importación, o configuración de Tailwind/Tsconfig que se rompa durante el proceso.
4. Ejecuta `npm run build` en el root final para simular el paso de Vercel y verificar que el código compila (Exit code 0).
5. Haz `git add .` y `git commit -m "fix: reconfigure repo root for native Vercel Next.js deployment"`.
</instructions>

<objective>
Configurar el proyecto en el directorio raíz para que al hacer `git push`, Vercel lo reconozca automáticamente como un proyecto Next.js y el despliegue funcione sin intervención manual.
</objective>
